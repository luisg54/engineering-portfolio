Overview

Summer research at WashU's Computational Mechanics & AI Lab under Dr. Fanwei Kong. The project focused on replacing a legacy Fortran-based cardiovascular solver with a modern, ML-extensible Python pipeline — and coupling it with a 3D CFD solver to simulate realistic cardiac hemodynamics.

Technical Work
  • Architected Physics-Informed Neural Networks (PINNs) in PyTorch that encode the governing Navier-Stokes equations directly into the loss function — achieving 1000× reduction in simulation time vs. traditional 3D CFD solvers
  • Built a modular Python pipeline to translate a legacy Fortran 0D cardiovascular solver — implementing Runge-Kutta 4 integrators, Newton iterations, and time-varying elastance and valve logic from scratch
  • Coupled the 0D circulation model with the 3D CFD solver via direct function calls — enabling physiologically accurate left atrial and ventricular pressure cycle reproduction
  • Validated cardiac valve pressure waveforms (MV, AV, TV, PV) against expected systolic/diastolic behavior; confirmed mass conservation across all valves
  • Presented findings at the WashU Summer Engineering Fellowship (WUSEF) symposium
Lessons Learned
Legacy code is documentation in disguise
The Fortran solver had no comments and no documentation. Reading it line by line to understand what each variable represented taught me more about cardiovascular modeling than any paper. Translating it forced a level of understanding that just using a modern library would have skipped entirely.
PINNs are powerful but not magic
The 1000× speedup is real but it comes with tradeoffs — PINNs struggle near discontinuities like valve opening/closing events. Understanding where the physics-based constraints actually help vs. where they fight the optimizer was the hardest part of the project.
Validation is the whole job
A fast solver that gives wrong answers is worse than a slow solver. Every result had to be checked against physiological reference data and conservation laws before it meant anything. I learned to be skeptical of my own outputs by default.
